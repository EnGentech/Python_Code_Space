Creating a Chatbot for Banking Tasks in Python

In this assignment, you will have the opportunity to practice your knowledge of Python functions by creating a chatbot that assists customers with banking tasks. Prior to this assignment, you will have learned about git, variables and data types, exception handling, looping and data structures. By completing this assignment, you will learn valuable skills that are essential for a professional software developer such as code organization, modularity, and best practices for developing functions and corresponding unit tests.

cd [PROJECT_NAME]

4. Connect the local repository to the remote repository on GitHub:

This step is about setting up a connection between your local Git repository and the remote repository on GitHub so you can push and pull changes between the two.

In the terminal/PowerShell, add the remote repository to your local repository:


git remote add origin https://github.com/[YOUR_GITHUB_USERNAME]/[PROJECT_NAME].git

Replace [YOUR_GITHUB_USERNAME] and [PROJECT_NAME] with the appropriate values.

The git remote add command is used to add the remote repository to your local repository.

5. Create a new file or use a provided file:

In the terminal/PowerShell, run the following command:


code .

The code . command is used to open the project directory in Visual Studio Code.

7. Stage and commit the changes:

After making changes to the code, you need to "stage" them for commit. This basically means telling Git which files you want to include in your next commit.

Return to the terminal/PowerShell and add the files to the staging area:


git add .

The git add command stages the changes you made to the README file.

After staging the changes, you can create a commit with a message describing the changes you made.

Commit the changes with a meaningful message:


git commit -m "Initial commit: Set up project directory and file structure"

8. Push the changes to the remote repository on GitHub:

This step is about pushing the changes you made to your local repository to the remote repository on GitHub.

Push the changes from your local repository to the remote repository on GitHub:


git push -u origin main

The git push command sends the changes to the remote repository under the name of the branch, in this case "main".

Here's how you can zip your project on Windows:

  1. Select all project files you want to include in the zip file.
  2. Right-click on one of the selected files and select "Send to" > "Compressed (zipped) folder".
  3. A new zipped folder will be created with the selected files. Rename the zipped folder to follow the naming convention specified above.

Preparing for Face-to-Face Reviews

Scroll to Table of Contents

To make the most of the limited time available for face-to-face reviews with instructors, it's important to come prepared. Follow these guidelines to ensure an efficient and productive review session:

  • Have your project's GitHub repository open: Before the review, ensure that you have pushed all your latest changes to the GitHub repository. Have the repository open in your browser, ready to show the instructor.
  • Open your code in Visual Studio Code: Have your project open in Visual Studio Code with all relevant files easily accessible. Familiarize yourself with the key parts of your code so you can quickly navigate and discuss it with the instructor.
  • Prepare a list of specific questions or concerns: If you have any doubts or need clarification on specific topics, prepare a list of questions or concerns beforehand. This will help you make the most of the limited time available during the review session.
  • Be ready to explain your code and thought process: Instructors may ask you about your code, the decisions you made, and your overall approach to the assignment. Be prepared to discuss your work and explain your reasoning.
  • Have a notepad and pen handy: Take notes during the review session to remember key points, feedback, and any follow-up actions you need to take.

Following these guidelines will help ensure a productive face-to-face review session and allow both you and your instructor to make the most of the time available.
